# Evals And Feedback

Use this reference to make the MVP measurable and improveable.

## Evaluation Layers

Use three layers:

1. Deterministic tests for ordinary software behavior.
2. AI behavior evals for quality, safety, and task success.
3. Customer feedback loops for real workflow value.

## Eval Set

Create examples for:

- happy path;
- confusing input;
- missing context;
- adversarial or unsafe input;
- edge cases from customer discovery;
- unacceptable output;
- gold-standard output.

## Feedback Loop

Capture:

- user goal;
- input;
- output;
- user correction;
- time saved or lost;
- failure category;
- requested next action.

## Output Format

```text
Success metric:
Eval cases:
Failure taxonomy:
Telemetry:
Feedback questions:
Review cadence:
Next iteration rule:
```
