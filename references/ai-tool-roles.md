# AI Tool Roles

The skill pack is provider-neutral. Use these roles to explain what kind of AI help a founder needs without requiring a specific vendor.

## Core Roles

| Role | Use |
| --- | --- |
| Chat assistant | Think through strategy, rewrite messaging, create checklists, critique assumptions. |
| Research assistant | Summarize markets, alternatives, customer language, accounts, and documents. |
| Coding agent | Implement scoped product tasks, tests, scripts, prototypes, and refactors. |
| Collaborative workspace | Maintain shared context, plans, docs, transcripts, and decisions. |
| Model/API platform | Power product features, evals, automations, and internal tools. |
| Workflow agent | Run recurring processes with human review and clear approval gates. |
| Evaluation assistant | Create test cases, compare outputs, classify failures, and monitor regressions. |

## Selection Rules

- Use the simplest role that can complete the job.
- Keep human review for commitments, money, security, hiring, and customer-facing claims.
- Prefer coding agents for bounded implementation tasks, not vague product ownership.
- Prefer workflow agents only after the founder understands the workflow manually.
- Prefer eval assistants before scaling AI behavior.

## Stage Fit

| Stage | Most Useful Roles |
| --- | --- |
| Idea | Chat assistant, research assistant, strategy critic. |
| MVP | Coding agent, architecture critic, evaluation assistant. |
| Launch | Sales copilot, feedback analyst, workflow agent. |
| Scale | Metrics analyst, ops agent, hiring assistant, fundraising copilot. |
